## 概述

### 系统架构

Taurus-AQuant 采用前后端分离架构：

- **前端**：Streamlit Web UI（端口 8501）
- **后端**：FastAPI 服务（端口 8000）
- **数据库**：PostgreSQL
- **缓存**：Redis

### 核心功能模块

1. **数据管理**：获取、存储、更新、导出数据
2. **策略管理**：创建、编辑、删除策略
3. **回测引擎**：执行回测，生成报告
4. **LLM Agent**：自然语言交互，策略生成
5. **实盘交易**：仿真交易和实盘交易

用户：帮我生成一个基于 MACD 指标的策略

Agent：好的，这是一个 MACD 策略：

```python
class MACDStrategy:
    def __init__(self, fast_period=12, slow_period=26, signal_period=9):
        self.fast_period = fast_period
        self.slow_period = slow_period
        self.signal_period = signal_period

    def generate_signals(self, data):
        # 计算 MACD
        exp1 = data['close'].ewm(span=self.fast_period, adjust=False).mean()
        exp2 = data['close'].ewm(span=self.slow_period, adjust=False).mean()
        data['macd'] = exp1 - exp2
        data['signal_line'] = data['macd'].ewm(span=self.signal_period, adjust=False).mean()

        # 生成信号
        data['signal'] = 0
        data.loc[data['macd'] > data['signal_line'], 'signal'] = 1  # 金叉买入
        data.loc[data['macd'] < data['signal_line'], 'signal'] = -1  # 死叉卖出

        return data
```

## 策略编写规范

### 策略模板

所有策略必须继承 `BaseStrategy` 类：

```python
# src/strategy/base_strategy.py

from abc import ABC, abstractmethod
import pandas as pd

class BaseStrategy(ABC):
    """策略基类"""

    def __init__(self, params: dict = None):
        self.params = params or {}

    @abstractmethod
    def generate_signals(self, data: pd.DataFrame) -> pd.DataFrame:
        """
        生成交易信号

        Args:
            data: 包含 'date', 'open', 'high', 'low', 'close', 'volume' 列的 DataFrame

        Returns:
            添加了 'signal' 列的 DataFrame
            signal = 1 表示买入
            signal = -1 表示卖出
            signal = 0 表示持有
        """
        pass

    def get_name(self) -> str:
        """获取策略名称"""
        return self.__class__.__name__

    def get_params(self) -> dict:
        """获取策略参数"""
        return self.params
```

### 策略示例：双均线策略

```python
# src/strategy/double_ma_strategy.py

import pandas as pd
from .base_strategy import BaseStrategy

class DoubleMAStrategy(BaseStrategy):
    """双均线策略"""

    def __init__(self, fast_period: int = 5, slow_period: int = 20):
        super().__init__({
            'fast_period': fast_period,
            'slow_period': slow_period
        })
        self.fast_period = fast_period
        self.slow_period = slow_period

    def generate_signals(self, data: pd.DataFrame) -> pd.DataFrame:
        """生成信号"""
        # 计算快慢均线
        data['fast_ma'] = data['close'].rolling(self.fast_period).mean()
        data['slow_ma'] = data['close'].rolling(self.slow_period).mean()

        # 生成信号
        data['signal'] = 0
        data.loc[data['fast_ma'] > data['slow_ma'], 'signal'] = 1  # 买入
        data.loc[data['fast_ma'] < data['slow_ma'], 'signal'] = -1  # 卖出

        return data
```

### 策略规范

1. **必须实现 `generate_signals` 方法**
2. **输入数据格式**：必须包含以下列
   - `date`: 日期
   - `open`: 开盘价
   - `high`: 最高价
   - `low`: 最低价
   - `close`: 收盘价
   - `volume`: 成交量

3. **输出数据格式**：必须添加 `signal` 列
   - `signal = 1`: 买入
   - `signal = -1`: 卖出
   - `signal = 0`: 持有

4. **不要修改原始数据**：使用 `data.copy()` 创建副本

5. **处理 NaN 值**：使用 `data.dropna()` 或填充

## 回测结果解读

### 核心指标说明

#### 1. 年化收益率 (Annual Return)

**定义**：年化后的平均收益率。

**计算公式**：
```
年化收益率 = (期末价值 / 期初价值)^(1/年数) - 1
```

**判断标准**：
- **优秀**：> 15%
- **良好**：10% - 15%
- **一般**：5% - 10%
- **较差**：< 5%

**注意**：必须考虑风险调整后收益（夏普比率）。

#### 2. 最大回撤 (Max Drawdown)

**定义**：策略历史最大亏损幅度。

**计算公式**：
```
最大回撤 = (峰值 - 谷值) / 峰值
```

**判断标准**：
- **优秀**：< 10%
- **良好**：10% - 20%
- **一般**：20% - 30%
- **较差**：> 30%

**注意**：最大回撤越大，恢复越困难。

#### 3. 夏普比率 (Sharpe Ratio)

**定义**：风险调整后收益，每承担一单位风险获得的超额收益。

**计算公式**：
```
夏普比率 = (年化收益率 - 无风险利率) / 年化波动率
```

**判断标准**：
- **优秀**：> 2.0
- **良好**：1.0 - 2.0
- **一般**：0.5 - 1.0
- **较差**：< 0.5

**注意**：夏普比率是评估策略优劣的核心指标。

#### 4. 胜率 (Win Rate)

**定义**：盈利交易占总交易的比例。

**计算公式**：
```
胜率 = 盈利交易次数 / 总交易次数
```

**判断标准**：
- **优秀**：> 60%
- **良好**：50% - 60%
- **一般**：40% - 50%
- **较差**：< 40%

**注意**：高胜率不代表高收益，还需看盈亏比。

#### 5. 盈亏比 (Profit/Loss Ratio)

**定义**：平均盈利与平均亏损的比值。

**计算公式**：
```
盈亏比 = 平均盈利金额 / 平均亏损金额
```

**判断标准**：
- **优秀**：> 2.0
- **良好**：1.5 - 2.0
- **一般**：1.0 - 1.5
- **较差**：< 1.0

**注意**：盈亏比和胜率需结合看。

#### 6. Alpha 和 Beta

**Alpha**：策略相对于基准的超额收益
- Alpha > 0：跑赢基准
- Alpha < 0：跑输基准

**Beta**：策略相对于基准的波动率
- Beta > 1：策略波动大于基准
- Beta < 1：策略波动小于基准

### 报告分析技巧

#### 1. 收益曲线分析

- **平滑度**：收益曲线越平滑越好
- **斜率**：斜率越大，收益越高
- **回撤期**：回撤期越短越好

#### 2. 持仓分析

- **持仓集中度**：单一持仓不超过 30%
- **持仓周期**：平均持仓周期是否符合策略类型
- **换手率**：换手率过高会增加交易成本

#### 3. 交易记录分析

- **交易频率**：避免过度交易
- **交易时机**：买入点和卖出点是否合理
- **手续费占比**：手续费不应超过总收益的 10%

## 实盘操作指南

### 仿真交易流程

#### 步骤 1：配置仿真账户

1. 点击 **"实盘配置"**
2. 选择 **"仿真账户"**
3. 设置初始资金（如 100 万元）
4. 点击 **"创建"**

#### 步骤 2：选择策略

1. 在策略列表中选择要通过仿真交易的策略
2. 配置策略参数
3. 点击 **"启动仿真"**

#### 步骤 3：监控运行

1. 在 **"实盘监控"** 页面查看运行状态
2. 查看持仓、订单、盈亏等信息
3. 如有异常，可以手动干预

#### 步骤 4：调整参数

如果策略表现不佳，可以：
- 调整策略参数
- 暂停策略
- 停止策略

### 实盘交易流程

**⚠️ 重要提醒**：
- 实盘交易有风险，可能导致资金损失
- 建议先在仿真环境充分测试（至少 3 个月）
- 小额资金灰度测试，逐步增加规模

#### 前提条件

1. **完成仿真测试**：仿真交易稳定运行 3 个月以上
2. **配置券商接口**：如 XTP
3. **配置风控参数**：止损、仓位、频率限制
4. **资金准备**：准备小额测试资金（如 1-5 万元）

#### 步骤 1：配置券商接口

1. 在券商开户并申请量化交易权限
2. 获取账户信息和密钥
3. 在 `.env` 文件中配置：

```bash
XTP_ACCOUNT=your_account
XTP_PASSWORD=your_password
XTP_SERVER_IP=120.27.164.138
XTP_SERVER_PORT=6001
```

4. 在 Web UI 中测试连接

#### 步骤 2：配置风控参数

1. 点击 **"风控配置"**
2. 设置以下参数：
   - **单只股票最大仓位**：如 20%
   - **总仓位上限**：如 80%
   - **单日最大交易次数**：如 10 次
   - **止损线**：如 -5%
   - **止盈线**：如 10%
   - **黑名单**：如 ST 股票

#### 步骤 3：启动实盘交易

1. 选择已通过仿真测试的策略
2. 点击 **"启动实盘"**
3. 系统提示风险警告，点击 **"确认"**
4. 输入二次确认密码
5. 策略开始执行

#### 步骤 4：日常监控

**每日检查**：
- 持仓情况
- 盈亏情况
- 风控触发情况

**每周检查**：
- 策略表现
- 市场环境变化
- 是否需要调整参数

**每月检查**：
- 月度绩效分析
- 风险评估
- 资金规模调整

### 紧急止损操作

如果市场出现极端情况，需要紧急止损：

#### 方法 1：Web UI

1. 在 **"实盘监控"** 页面
2. 点击 **"紧急止损"** 按钮
3. 系统自动卖出所有持仓
4. 发送通知给用户

#### 方法 2：API

```bash
curl -X POST "http://localhost:8000/api/trading/emergency_stop" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### 方法 3：手动操作

1. 登录券商 APP
2. 手动卖出所有持仓

## 常见问题

### 1. 回测速度太慢怎么办？

**原因**：数据量过大或策略计算复杂。

**解决方案**：
- 缩短回测时间范围
- 优化策略代码，使用向量化计算
- 升级服务器配置

### 2. Agent 生成的策略无法运行怎么办？

**原因**：LLM 生成的代码可能有语法错误或逻辑错误。

**解决方案**：
- 检查代码语法
- 查看 Agent 提示的错误信息
- 手动修改代码
- 向 Agent 提供更详细的描述

### 3. 数据下载失败怎么办？

**原因**：网络问题或 API 限制。

**解决方案**：
- 检查网络连接
- 检查 Tushare Token 是否正确
- 稍后重试
- 切换到 AkShare 数据源

### 4. 实盘交易下单失败怎么办？

**原因**：风控限制或券商接口问题。

**解决方案**：
- 检查风控参数
- 查看券商接口日志
- 检查账户余额
- 联系券商技术支持
