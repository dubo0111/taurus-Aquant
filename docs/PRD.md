## 一、产品概述

### 1.1 产品愿景

**让每个人都能通过 AI 技术降低量化交易的技术门槛，实现智能化投资。**

### 1.2 产品定位

Taurus-AQuant 是一个 **面向个人投资者的 A股/ETF 算法交易平台**，核心特点是：

- **易用性**：15 分钟快速启动，零编程基础也能使用基础功能
- **专业性**：完整的回测引擎、风控系统、绩效分析
- **智能化**：集成大语言模型（LLM），支持自然语言交互
- **开放性**：开源社区驱动，支持自定义扩展

### 1.3 核心价值主张

| 价值主张 | 描述 | 目标用户 |
|----------|------|----------|
| **降低技术门槛** | Docker 一键启动，无需复杂环境配置 | 量化新手 |
| **AI 赋能决策** | LLM 辅助策略开发、风险管理、市场分析 | AI 爱好者 |
| **专业回测** | 完整的中国市场特性支持（停牌、复权、费用） | 个人投资者 |
| **实盘预备** | 仿真交易 → 小额实盘灰度 → 生产级实盘 | 认真投资者 |
| **开放协作** | 开源项目，社区驱动，持续迭代 | 开发者 |

### 2.2 非目标用户

- **机构投资者**：他们有专业的团队和系统，本项目功能过于简单
- **高频交易者**：本项目不支持毫秒级交易
- **期货/期权交易者**：本项目初期只支持 A股/ETF
- **期望"稳赚不赔"的用户**：量化交易同样有风险，不适合风险厌恶者
- **完全不懂电脑的用户**：至少需要基本的电脑操作能力

### 3.2 模块 1：数据管理模块 (P0)

#### 3.2.1 功能概述

提供 A股/ETF 历史数据和实时数据的获取、存储、更新、导出功能。

#### 3.2.2 功能列表

| 功能点 | 优先级 | 描述 | 验收标准 |
|--------|--------|------|----------|
| **数据源接入** | P0 | 支持 AkShare 和 Tushare 两种数据源 | 能获取日线、分钟线数据 |
| **自动更新** | P0 | 每日自动更新最新数据 | 每日 18:00 自动更新 |
| **数据质量检查** | P1 | 检查数据缺失、异常值 | 自动标记异常数据 |
| **交易日历** | P0 | 提供中国交易日历服务 | 准确识别交易日和非交易日 |
| **复权处理** | P0 | 自动处理前复权、后复权 | 复权结果与主流软件一致 |
| **停牌处理** | P1 | 识别停牌期间，标记数据 | 停牌期间数据标记为 NaN |
| **数据导出** | P1 | 支持导出 CSV/Excel/Parquet | 导出数据可被其他工具使用 |
| **缓存机制** | P0 | 使用 Redis 缓存热点数据 | 缓存命中率 > 80% |

#### 3.2.3 用户故事

**US-1.1**: 作为一个用户，我想要下载某只股票的历史数据，以便进行策略研究。

- **前置条件**：用户已配置 Tushare Token
- **操作步骤**：
  1. 用户在 Web UI 输入股票代码（如 000001.SZ）
  2. 选择数据类型（日线/分钟线）
  3. 选择时间范围（如 2020-01-01 至 2023-12-31）
  4. 点击"下载数据"
  5. 系统从 Tushare 获取数据并存储到数据库
- **验收标准**：
  - 数据下载成功率 > 95%
  - 下载速度 < 5 秒/年数据
  - 数据完整率 > 99%

**US-1.2**: 作为一个用户，我想要自动更新每日数据，以便及时跟踪市场。

- **前置条件**：系统已启动，数据库已配置
- **操作步骤**：
  1. 系统在每日 18:00 自动检查是否有新数据
  2. 如果有新数据，自动下载并更新数据库
  3. 发送更新通知（邮件/微信）
- **验收标准**：
  - 自动更新成功率 > 90%
  - 数据延迟 < 1 天

### 3.3 模块 2：回测引擎模块 (P0)

#### 3.3.1 功能概述

提供完整的回测功能，包括策略执行、绩效分析、报告生成。

#### 3.3.2 功能列表

| 功能点 | 优先级 | 描述 | 验收标准 |
|--------|--------|------|----------|
| **回测引擎** | P0 | 集成 Backtrader/Rqalpha | 支持完整的回测流程 |
| **多周期回测** | P1 | 支持日线、分钟线、Tick 级回测 | 至少支持日线和分钟线 |
| **多品种回测** | P1 | 支持股票、ETF 组合回测 | 支持至少 5 个品种组合 |
| **费用模型** | P0 | 佣金、印花税、过户费 | 费用计算准确率 100% |
| **绩效分析** | P0 | 年化收益、夏普比率、最大回撤等 | 至少 10 个核心指标 |
| **归因分析** | P2 | Brison 归因、风险归因 | 可选功能 |
| **报告生成** | P0 | 生成 HTML/PDF 报告 | 报告包含图表和详细说明 |
| **可视化** | P0 | 收益曲线、持仓变化、回撤图 | 图表清晰美观 |

#### 3.3.3 用户故事

**US-2.1**: 作为一个用户，我想要对某个策略进行回测，以便评估其历史表现。

- **前置条件**：用户已有策略代码或选择内置策略
- **操作步骤**：
  1. 用户在 Web UI 选择策略（如"双均线策略"）
  2. 配置回测参数（股票代码、时间范围、初始资金、策略参数）
  3. 点击"开始回测"
  4. 系统执行回测，显示进度
  5. 回测完成后，显示绩效报告
- **验收标准**：
  - 回测执行时间 < 1 分钟（1 年日线数据）
  - 绩效指标计算准确
  - 报告清晰易读

**US-2.2**: 作为一个用户，我想要对比不同策略的回测结果，以便选择最佳策略。

- **前置条件**：用户已完成至少 2 个策略的回测
- **操作步骤**：
  1. 用户在"结果对比"页面选择多个回测结果
  2. 系统并排显示多个策略的绩效指标
  3. 显示收益曲线对比图
- **验收标准**：
  - 对比视图清晰直观
  - 关键指标差异明显

### 3.4 模块 3：LLM Agent 模块 (P1)

#### 3.4.1 功能概述

集成大语言模型，支持自然语言交互、策略生成、风险分析等功能。

#### 3.4.2 功能列表

| 功能点 | 优先级 | 描述 | 验收标准 |
|--------|--------|------|----------|
| **自然语言交互** | P1 | 支持自然语言问答 | 80% 问题能准确回答 |
| **策略生成** | P1 | 自然语言生成策略代码 | 生成的代码能通过语法检查 |
| **策略优化** | P2 | 提供策略优化建议 | 建议合理且可执行 |
| **风险分析** | P1 | 识别策略风险点 | 覆盖主要风险类型 |
| **市场解读** | P2 | 解读市场新闻和事件 | 可选功能 |
| **多模型支持** | P1 | 支持 OpenAI/Claude/本地 LLM | 至少支持 2 种模型 |
| **Prompt 管理** | P1 | 管理和优化 Prompt 模板 | Prompt 库可扩展 |

#### 3.4.3 用户故事

**US-3.1**: 作为一个用户，我想要用自然语言生成一个策略，以便快速实现我的想法。

- **前置条件**：用户已配置 OpenAI API Key
- **操作步骤**：
  1. 用户在 Agent 对话框输入："帮我生成一个基于 MACD 指标的策略"
  2. Agent 理解意图，生成策略代码
  3. 用户查看生成的代码，可以修改参数
  4. 用户点击"保存策略"
  5. 系统保存策略到策略库
- **验收标准**：
  - 生成代码语法正确
  - 策略逻辑基本合理
  - 响应时间 < 10 秒

**US-3.2**: 作为一个用户，我想要 Agent 帮我检查策略的风险点，以便改进策略。

- **前置条件**：用户已有一个策略
- **操作步骤**：
  1. 用户选择某个策略，点击"风险分析"
  2. Agent 分析策略代码和历史表现
  3. Agent 列出主要风险点（如过度拟合、仓位过重等）
  4. Agent 提供改进建议
- **验收标准**：
  - 识别至少 3 个主要风险点
  - 建议具体可行
